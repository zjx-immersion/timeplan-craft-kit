# V11.3 用户指南 - 如何查看更改

## 重要说明

之前的V11.3更改分为两部分：

### 第1部分：版本对比视图（VersionTableView）✅
- **位置**: 点击工具栏 "版本" 按钮（历史图标）
- **更改内容**:
  - "任务"重命名为"计划单元"
  - 移除"状态"标签列
  - 保留类型、依赖关系、时长核心列
- **状态**: ✅ 已完成

### 第2部分：表格视图（TableView）✅ NEW
- **位置**: 点击工具栏 "表格" 按钮（表格图标）
- **更改内容**:
  - 新增"类型"列（计划单元/里程碑/关口）
  - 新增"时长"列（天数）
  - 新增"依赖关系"列
  - 名称列添加类型图标
  - 移除"标签"和"优先级"列
- **状态**: ✅ 本次提交完成

### 第3部分：OrionX数据增强（v4）✅
- **位置**: 在甘特图或任何视图中查看
- **更改内容**:
  - E0架构拆分为3个MR
  - 视觉感知拆分为4个MR
  - 行为决策拆分为3个MR
  - 路径规划拆分为2个MR
  - 新增12+个依赖关系
- **状态**: ✅ 已完成

## 如何查看V11.3的更改？

### 步骤1: 确保查看正确的计划
1. 打开应用: http://localhost:9082
2. 在项目列表中选择 **"Orion X 智能驾驶平台 2026 年度计划（完整版）"**
3. 确认ID为 `orion-x-2026-full-v3` 或包含 v4 数据

### 步骤2: 查看表格视图（本次新增）
1. 点击工具栏的 **"表格"** 按钮（TableOutlined图标）
2. 查看新增的列：
   - **类型**列：显示计划单元/里程碑/关口标签
   - **时长**列：显示任务时长（天数）
   - **依赖关系**列：显示前置依赖任务
3. 名称列现在包含类型图标
4. 标签和优先级列已移除

### 步骤3: 查看版本对比视图
1. 点击工具栏的 **"版本"** 按钮（HistoryOutlined图标）
2. 选择两个版本进行对比
3. 查看优化后的列布局

### 步骤4: 查看甘特图中的MR拆解
1. 保持在 **甘特图** 视图（BarChartOutlined图标）
2. 查看以下Timeline的MR拆解：
   - **电子电器架构**: E0.1域控架构 → E0.2传感器配置 → E0.3系统通信
   - **感知算法**: P1.1预处理 → P1.2检测 → P1.3跟踪 → P1.4语义
   - **规划决策**: PL1.1状态机 → PL1.2强化学习 → PL1.3评估
   - **路径规划**: PL2.1全局规划 → PL2.2局部规划
3. 查看MR之间的依赖连线（箭头）

## 刷新浏览器

如果看不到更改，请：
1. **硬刷新**: Cmd+Shift+R (Mac) 或 Ctrl+Shift+R (Windows)
2. **清除缓存**: 
   - Chrome: 开发者工具 → Network → Disable cache
   - 或在设置中清除浏览数据
3. **重启开发服务器**:
   ```bash
   # Ctrl+C 停止
   npm run dev
   ```

## 视图切换快捷键

| 视图 | 图标 | 描述 |
|-----|------|------|
| 甘特图 | BarChartOutlined | 时间线可视化，显示依赖关系 |

## 常见问题

### Q: 为什么我看到的表格没有变化？
A: 请确认：
1. 是否查看的是正确的计划（OrionX v3/v4）
2. 是否点击了正确的视图按钮（"表格"或"版本"）
3. 是否已硬刷新浏览器（Cmd+Shift+R）

### Q: 迭代视图中为什么看不到依赖连线？
A: 迭代视图当前不显示依赖关系连线，请在**甘特图视图**中查看。我们计划在后续版本中为迭代视图添加依赖关系可视化。

### Q: OrionX数据的MR在哪里？
A: MR数据已包含在 `orion-x-2026-full-v3` 计划中。在甘特图中，每个Timeline会显示多个子任务节点（如E0.1、E0.2等）。
